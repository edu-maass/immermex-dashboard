Documentación Funcional - Dashboard Web
Immermex
Este documento describe los requerimientos funcionales y técnicos del Dashboard Web de Immermex,
con el objetivo de guiar al equipo de desarrollo en la implementación. Incluye los indicadores a
generar, la dinámica de cálculo, así como los elementos de interfaz de usuario necesarios para cumplir
con los requerimientos de dirección.
Indicadores Clave (KPIs)
- Facturación Total (periodo).
- Cobranza Total (periodo).
- % Cobranza sobre Facturación.
- Cartera vencida (aging: 0-30, 31-60, 61-90, 90+).
- Días de Cuentas por Cobrar (clásico vs ajustado por anticipos).
- Anticipos recibidos (% de facturación cubierta y días anticipofifactura).
- Rotación de Inventarios (días cortefifactura).
- Consumo por material (kg, importe, margen).
- Análisis por pedido (importe facturado, kg, ticket promedio, margen, estado de cobro).
- Ciclo de Conversión de Efectivo (Inventario + CxC).
- Top clientes: ranking por facturación, puntualidad y rentabilidad.
Dinámica de Cálculo
1. Facturación: se obtiene de la hoja 'facturacion' (campos neto, total, fecha, cliente, agente). 2.
Cobranza: se cruza con facturación usando folio/UUID; se calculan días reales vs pactados. 3.
Anticipos: se consideran como cobro inmediato (0 días) sobre el monto recibido; se recalculan días de
CxC como promedio ponderado. 4. Inventarios: se calcula tiempo entre fecha de corte y fecha de
factura; se obtiene rotación. 5. Consumo por material: se agrupa por código de material y pedido; se
calcula kg, importe y margen (requiere catálogo de costos). 6. Pedidos: seleccionando un número de
pedido se muestra detalle de materiales, facturas, estado de cobro y rentabilidad. 7. KPIs se recalculan
automáticamente al cambiar filtros de periodo, cliente o pedido.
Requerimientos de Interfaz de Usuario
- Pantalla principal: resumen ejecutivo con KPIs en tarjetas (facturación, cobranza, % cobrado, ciclo de
efectivo).
- Gráficos dinámicos: barras para top clientes, aging de cartera, consumo por material.
- Filtros interactivos (slicers): por periodo, cliente, agente, número de pedido, material.
- Vista por pedido: al seleccionar un número de pedido se muestra detalle de facturas, materiales, kg,
importe, margen y cobro.
- Vista por cliente: resumen integral del cliente (facturación, cobros, anticipos, puntualidad, ticket
promedio).
- Exportación: opción de exportar tablas a Excel o CSV para análisis detallado.
- Actualización mensual: dashboard debe permitir carga del archivo mensual estandarizado y refrescar
todos los indicadores automáticamente.
Experiencia de Usuario Esperada
El dashboard debe ser intuitivo, con navegación simple. El usuario de dirección debe poder obtener en
máximo 3 clics cualquier respuesta clave: ¿Cuánto hemos facturado? ¿Cuánto hemos cobrado?
¿Quién debe más? ¿Cuál es el estatus de un pedido específico? ¿Qué material está generando más
margen?
